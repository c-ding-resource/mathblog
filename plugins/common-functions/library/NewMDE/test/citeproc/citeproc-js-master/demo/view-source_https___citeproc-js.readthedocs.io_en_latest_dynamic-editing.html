<div class="section" id="dynamic-editing">
<h2>Dynamic Editing<a class="headerlink" href="#dynamic-editing" title="Permalink to this headline"></a></h2>
<p><strong>Style:</strong> <select id="citation-styles"></select></p>
<p>This page demonstrates dynamic citation editing in the browser. The
underlying code was prepared in response to work by <a class="reference external" href="https://github.com/dsifford">Derek Sifford</a>. Click on the chevrons to open a
citation widget,&nbsp;<span class="octicon octicon-chevron-down citeme" title="Edit citations"></span> select one or more references, and press
Save to add them to the document.&nbsp;<span class="octicon octicon-chevron-down citeme" title="Edit citations"></span> Use the pulldown list
above to transform the document to another style.&nbsp;<span class="octicon octicon-chevron-down citeme" title="Edit citations"></span> Click on
the More button below for information on running the code locally,
and on adapting it for use in production.&nbsp;<span class="octicon octicon-chevron-down citeme" title="Edit citations"></span></p>

<div class="section" id="running-locally">
<h2>Running locally<a class="headerlink" href="#running-locally" title="Permalink to this headline"></a></h2>
<p>One of the main purposes of this page is to provide a worked example
for developers. A good way to explore the way it all works is to
run the page locally. Here is how to set that up.</p>
<div class="section" id="requirements">
<h3>Requirements<a class="headerlink" href="#requirements" title="Permalink to this headline"></a></h3>
<ul class="simple">
<li><a class="reference external" href="https://git-scm.com/">git</a></li>
<li><a class="reference external" href="https://nodejs.org/en/">node.js</a></li>
<li><a class="reference external" href="https://www.npmjs.com/">npm</a></li>
<li><a class="reference external" href="http://www.sphinx-doc.org/en/stable/">Sphinx</a></li>
</ul>
</div>
<div class="section" id="local-setup">
<h3>Local setup<a class="headerlink" href="#local-setup" title="Permalink to this headline"></a></h3>
<dl class="docutils">
<dt>Fetch the repo</dt>
<dd><p class="first">Clone the <code class="docutils literal notranslate"><span class="pre">citeproc-js</span></code> documentation project and enter
its top-level directory:</p>
<div class="last highlight-bash notranslate"><div class="highlight"><pre><span></span>git clone https://github.com/Juris-M/citeproc-js-docs.git --recursive
<span class="nb">cd</span> citeproc-js-docs
</pre></div>
</div>
</dd>
<dt>Build the docs</dt>
<dd><p class="first">The following command should work:</p>
<div class="last highlight-bash notranslate"><div class="highlight"><pre><span></span>make html
</pre></div>
</div>
</dd>
<dt>Run a server using <code class="docutils literal notranslate"><span class="pre">node.js</span></code></dt>
<dd><p class="first">The built page uses <code class="docutils literal notranslate"><span class="pre">XMLHttpRequest()</span></code> calls, so it must be
viewed through a web server. To run a simple server using
<code class="docutils literal notranslate"><span class="pre">node.js</span></code>, an incantation like this should do the trick:</p>
<div class="last highlight-bash notranslate"><div class="highlight"><pre><span></span>npm install -g http-server
http-server _build/html
</pre></div>
</div>
</dd>
</dl>
</div>
</div>
<div class="section" id="source-file-overview">
<h2>Source File Overview<a class="headerlink" href="#source-file-overview" title="Permalink to this headline"></a></h2>
<dl class="docutils">
<dt><code class="docutils literal notranslate"><span class="pre">_static/js/citeproc.js</span></code></dt>
<dd>The <span style="background:#cccccc;border-radius:0.5em;padding-left:0.25em;padding-right:0.25em;font-weight:bold;font-family:'Courier New', Courier, monospace">citeproc-js</span> <a class="reference external" href="https://citationstyles.org">CSL</a> processor for Citation Style Language (CSL) stylesheets.</dd>
<dt><code class="docutils literal notranslate"><span class="pre">_static/js/citesupport-es5.js</span></code></dt>
<dd>A middleware module that connects the processor, which runs in a web worker, with the DOM of the page.</dd>
<dt><code class="docutils literal notranslate"><span class="pre">_static/js/citeworker.js</span></code></dt>
<dd>A web worker to run the processor. This
supplies data I/O methods required by the processor, and implements
a simple messaging interface for purposes of the demo page.</dd>
<dt><code class="docutils literal notranslate"><span class="pre">_static/css/screen.css</span></code></dt>
<dd>CSS code for the <span style="background:#cccccc;border-radius:0.5em;padding-left:0.25em;padding-right:0.25em;font-weight:bold;font-family:'Courier New', Courier, monospace">citeproc-js</span> documentation, including
the demo pages.</dd>
<dt><code class="docutils literal notranslate"><span class="pre">_static/data/items</span></code></dt>
<dd>A few sample items for the editing demo, in CSL JSON format.</dd>
<dt><code class="docutils literal notranslate"><span class="pre">_static/data/locales</span></code></dt>
<dd>The <a class="reference external" href="https://github.com/citation-style-language/locales">standard CSL locales</a>.</dd>
<dt><code class="docutils literal notranslate"><span class="pre">_static/data/styles</span></code></dt>
<dd>The CSL styles used in the demo. The JM styles are from the
<a class="reference external" href="https://github.com/juris-m/jm-styles">Juris-M styles repository</a>, and have modular legal
style support. The remainder are from the <a class="reference external" href="https://github.com/citation-style-language/styles/">official CSL repository</a>,
which feeds the <a class="reference external" href="https://www.zotero.org/styles">Zotero styles</a> distribution site.</dd>
<dt><code class="docutils literal notranslate"><span class="pre">_static/data/juris</span></code></dt>
<dd>A set of legal style modules resides here. For information on legal
style support in CSL-M (multilingual-enhanced CSL) see the
<a class="reference external" href="https://juris-m.github.io/editor/">Juris-M Style Editor</a>
(GitHub account required).</dd>
</dl>
</div>
<div class="section" id="integrator-notes">
<h2>Integrator notes<a class="headerlink" href="#integrator-notes" title="Permalink to this headline"></a></h2>
<p>Here are some notes on things relevant to deployment:</p>
<ul class="simple">
<li>The class should be instantiated as <code class="docutils literal notranslate"><span class="pre">citesupport</span></code>. The event
handlers expect the class object to be available in global
context under that name.</li>
<li>If <code class="docutils literal notranslate"><span class="pre">config.demo</span></code> is <code class="docutils literal notranslate"><span class="pre">true</span></code>, the stored object <code class="docutils literal notranslate"><span class="pre">citationIdToPos</span></code>
maps citationIDs to the index position of fixed pegs in the
document that have class <code class="docutils literal notranslate"><span class="pre">citeme</span></code>. In the demo, this map is
stored in localStorage, and is used to reconstruct the document
state (by reinserting <code class="docutils literal notranslate"><span class="pre">class:citation</span></code> span tags) on page reload.</li>
<li>If <code class="docutils literal notranslate"><span class="pre">config.demo</span></code> is <code class="docutils literal notranslate"><span class="pre">false</span></code>, the document is assumed to contain
<code class="docutils literal notranslate"><span class="pre">class:citation</span></code> span tags, and operations on <code class="docutils literal notranslate"><span class="pre">citeme</span></code> nodes will
not be performed. In non-demo mode, <code class="docutils literal notranslate"><span class="pre">citationIdToPos</span></code> carries
the index position of citation nodes for good measure, but the
mapping is not used for anything.</li>
<li>The <code class="docutils literal notranslate"><span class="pre">spoofDocument()</span></code> function brings citation data into memory.
In the demo, this data is held in localStorage, and
<code class="docutils literal notranslate"><span class="pre">spoofDocument()</span></code> performs some sanity checks on data and
document. For a production deployment, this is the place for code
that initially extracts citation data the document (if, for example,
it is stashed in data-attributes on citation nodes).</li>
<li>The <code class="docutils literal notranslate"><span class="pre">setCitations()</span></code> function is where citation data for individual
citations would be saved, at the location marked by NOTE.</li>
<li>The user-interface functions <code class="docutils literal notranslate"><span class="pre">buildStyleMenu()</span></code> and
<code class="docutils literal notranslate"><span class="pre">citationWidget()</span></code> are simple things cast for the demo, and
should be replaced with something a bit more functional.</li>
<li>The <code class="docutils literal notranslate"><span class="pre">SafeStorage</span></code> class should be replaced (or subclassed?) for
deployment with a class that provides the same methods. If
the citation objects making up <code class="docutils literal notranslate"><span class="pre">citationByIndex</span></code> are stored
directly on the <code class="docutils literal notranslate"><span class="pre">class:citation</span></code> span nodes, the getter for
that value should harvest the values from the nodes, and
store them on <code class="docutils literal notranslate"><span class="pre">config.citationByIndex</span></code>. The setter should
set <code class="docutils literal notranslate"><span class="pre">config.citationByIndex</span></code> only, relying on other code
to update the node value.</li>
<li>Probably some other stuff that Ive overlooked.</li>
</ul>
</div>
<div class="section" id="worker-api">
<h2>Worker API<a class="headerlink" href="#worker-api" title="Permalink to this headline"></a></h2>
<p>The heavy lifting is done by the CSL processor, which runs in a
separate thread as a web worker. Only the document-facing interface of
the worker is described here: it should not be necessary to tangle
with the internals of the worker itself. Its only idiosyncracy is that
it assigns note numbers (reflected in the return) in citation
sequencein contrast to word processor context, it assumes that the
only footnotes in the document are those generated automatically by a
note style. If that is not true in your context, you will want to
disable that behavior, and do whatever is necessary on document side
to extract real note numbers for delivery to the processor.</p>
<p>The worker is controlled by two methods, <code class="docutils literal notranslate"><span class="pre">callInitProcessor()</span></code> and
<code class="docutils literal notranslate"><span class="pre">callRegisterCitation()</span></code>, each with a corresponding message and
return event.</p>
<dl class="docutils">
<dt><code class="docutils literal notranslate"><span class="pre">citesupport.callInitProcessor(styleID,</span> <span class="pre">localeID)</span></code></dt>
<dd><p class="first">This method is used on page load, on change of style, and when all
citations have been removed from the document. The <code class="docutils literal notranslate"><span class="pre">styleID</span></code>
argument is mandatory. If <code class="docutils literal notranslate"><span class="pre">localeID</span></code> is not provided, the
processor will be configured with the <code class="docutils literal notranslate"><span class="pre">en-US</span></code> locale.</p>
<p>The <code class="docutils literal notranslate"><span class="pre">citesupport.callInitProcessor</span></code> method implicitly accesses the
<code class="docutils literal notranslate"><span class="pre">config.citationByIndex</span></code> array, which must be accessible in page
context. If the array is empty, the processor will be initialized
without citations. If the array contains citations, the processor
will be initialized to that document state, and return an array of
arrays as <code class="docutils literal notranslate"><span class="pre">rebuildData</span></code>, for use in reconstructing citations in
the document text. Each sub-array contains a citation ID, a note
number, and a citation string. For example, if the <code class="docutils literal notranslate"><span class="pre">styleID</span></code> is
for a <code class="docutils literal notranslate"><span class="pre">note</span></code> style, and if <code class="docutils literal notranslate"><span class="pre">config.citationByIndex</span></code> yields the
citations Wurzel Gummidge (1990) and My Aunt Sally (2001), the
<code class="docutils literal notranslate"><span class="pre">rebuildData</span></code> structure would look like this:</p>
<div class="last highlight-javascript notranslate"><div class="highlight"><pre><span></span><span class="p">[</span>
<span class="p">[</span>
<span class="s2">&quot;lu7Tu3ki&quot;</span><span class="p">,</span>
<span class="s2">&quot;1&quot;</span><span class="p">,</span>
<span class="s2">&quot;Wurzel Gummidge (1990)&quot;</span>
<span class="p">],</span>
<span class="p">[</span>
<span class="s2">&quot;ko4aNoo9&quot;</span><span class="p">,</span>
<span class="s2">&quot;2&quot;</span><span class="p">,</span>
<span class="s2">&quot;My Aunt Sally (2001)&quot;</span>

<span class="p">]</span>
<span class="p">]</span>
</pre></div>
</div>
</dd>
<dt><code class="docutils literal notranslate"><span class="pre">citesupport.callRegisterCitation(citation,</span> <span class="pre">preCitations,</span> <span class="pre">postCitations)</span></code></dt>
<dd><p class="first">This method is used to add or to edit citations. All three
arguments are mandatory. <code class="docutils literal notranslate"><span class="pre">citation</span></code> is an ordinary citation
object as described above. <code class="docutils literal notranslate"><span class="pre">preCitations</span></code> and <code class="docutils literal notranslate"><span class="pre">postCitations</span></code>
are arrays of arrays, in which each sub-array is composed of a
citation ID and a note number. For example, if a note citation
is to be inserted between the Wurzel Gummidge and Aunt Sally
citations in the example above, these would have the following form:</p>
<div class="highlight-javascript notranslate"><div class="highlight"><pre><span></span><span class="nx">preCitations</span> <span class="o">=</span> <span class="p">[</span>
<span class="p">[</span>
<span class="s2">&quot;lu7Tu3ki&quot;</span><span class="p">,</span>
<span class="s2">&quot;1&quot;</span>
<span class="p">]</span>
<span class="p">];</span>

<span class="nx">postCitations</span> <span class="o">=</span> <span class="p">[</span>
<span class="p">[</span>
<span class="s2">&quot;ko4aNoo9&quot;</span><span class="p">,</span>
<span class="s2">&quot;3&quot;</span>
<span class="p">]</span>
<span class="p">];</span>
</pre></div>
</div>
<p>Notice the change to the note number: the processor registers
note numbers for use in back-references, but maintenance of
correct note numbering must be handled in document-side code.</p>
<p>The <code class="docutils literal notranslate"><span class="pre">citesupport.callRegisterCitation</span></code> method returns two values from the
processor: <code class="docutils literal notranslate"><span class="pre">citationByIndex</span></code> (described above) and <code class="docutils literal notranslate"><span class="pre">citations</span></code>.
The latter is an array of one or more arrays, each composed of a
citation position index, a string, and a citation ID. For example,
the return value to insert a citation Calvin (1995); Hobbes
(2016) between the Wurzel Gummidge and My Aunt Sally citations
would look something like this:</p>
<div class="highlight-javascript notranslate"><div class="highlight"><pre><span></span><span class="p">[</span>
<span class="p">[</span>
<span class="mi">1</span><span class="p">,</span>
<span class="s2">&quot;Calvin (1995); Hobbes (2016)&quot;</span><span class="p">,</span>
<span class="s2">&quot;Ith7eg8T&quot;</span>
<span class="p">]</span>
<span class="p">]</span>
</pre></div>
</div>
<p class="last">Note that the return value might contain updates for multiple
citations.</p>
</dd>
</dl>
